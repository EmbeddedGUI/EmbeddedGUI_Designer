# UI 极简化改造计划（Android Studio 风格）

## 目标

将当前编辑器 UI 回归到“工具型界面”：

- 文本优先，减少图标依赖（有文字就不放图标）
- 去装饰化，强化信息结构与操作效率
- 稳定布局与低干扰视觉反馈

---

## 当前阶段（Phase 1）

### 已完成（Phase 1 + Phase 2 首批）

1. `project_dock.py`
   - 移除 `New Page` 按钮图标，仅保留文字。
   - 移除页面树中每行的图标，改为纯文本层级表达。
   - 收敛标题文案，降低“营销式”文案密度。

2. `resource_panel.py`
   - 移除图片/字体/文本/字符串模块中按钮图标，统一文字按钮。
   - 移除资源 Tab 图标（Images/Fonts/Text/Strings 均为文字标签）。
   - 保留原有功能路径与可访问性描述，不改业务逻辑。

3. `typography.py`
   - 补充注释，明确“IDE 风格紧凑层级”的目标。
   - 新增 `status` 角色，为状态文本提供统一入口。

4. `theme.py`（Phase 2 首批）
   - 收紧全局 token：按钮/输入 padding 下调，Tab 最小高度下调。
   - 收敛字体层级：`h1/h2/panel_title/body/body_sm` 统一下调 1px。
   - 收敛圆角：`r_md/r_xl` 从偏展示型改为工具型尺度。
   - 去除 `project/resource/property/workspace command` 等关键区块渐变，改为纯色面板。

---

## 进行中事项（Phase 2）

### 1) 样式 Token 收敛

- 统一面板间距到 4pt 系统：4/8/12/16
- 统一标题层级（仅 2~3 级）
- 统一按钮高度/边距，避免不同区域风格漂移

### 2) 去“卡片化”和强装饰

- 减少 header/metric 区块视觉强调
- 降低阴影、圆角、渐变或不必要分割
- 保留必要状态色（选中/警告/错误）

### 3) 信息密度优化

- 列表行高稳定在 28~32px
- 减少重复说明文案，更多通过结构表达
- 状态信息集中在状态条或摘要区，避免多处重复

---

## 验收标准

1. **一致性**：
   - 同类控件（按钮、列表、tab）在各面板视觉一致。

2. **简约性**：
   - 默认视图中，图标仅保留“不可替代”的极少数状态图标。

3. **可读性**：
   - 主流程（页面管理、资源管理、属性编辑）无需图标也能快速理解。

4. **低干扰**：
   - 无多余动画；仅保留必要 hover / selected 反馈。

---

## 风险与回滚策略

- 风险：用户对旧视觉“熟悉路径”短期不适应。
- 策略：
  - 每个阶段保持功能不变，仅视觉收敛；
  - 若出现可用性退化，按模块回滚（Project Dock / Resource Panel 分开回滚）。

---

## 跟进节奏建议

- 每周一次 UI 巡检：
  - 检查新增控件是否遵守“文字优先、低装饰、统一间距”
  - 对偏离项建立小修复清单，避免再次风格发散
