# 画布编辑

Designer 的核心体验，最终还是体现在画布编辑上。

## 画布里最常见的动作

日常使用里，高频操作通常是：

- 选中控件
- 拖动控件
- 调整尺寸
- 多选控件
- 对齐和分布
- 缩放画布

## 单选与多选

建议这样理解：

- 单选：主要看和改一个控件的属性
- 多选：主要做对齐、分布、批量属性修改

如果你发现右侧属性区看起来“不像单个控件”，通常说明当前是多选状态。

## 对齐与分布什么时候用

下面两类场景特别适合：

- 多个按钮、图标或卡片要做整齐排列
- 页面做完后要统一收口视觉间距

如果控件已经由布局容器管理，再强行做自由对齐反而容易造成混乱。

## 网格与吸附

在 `View` 菜单里可以控制：

- `Show Grid`
- `Grid Size`

建议新手默认保持网格开启，因为它能明显减少“看着差不多，其实没对齐”的问题。

## Focus Canvas 的适用场景

当你需要专注看画布本身时，可以打开：

- `View -> Focus Canvas`

这个模式会尽量压缩左右和底部干扰区域，适合做最终排版微调。

## 背景稿的使用

如果你手里有 UI 设计稿图片，可以把它作为背景 mockup 导入当前页面，对照摆控件会更直观。

但要注意：

- 背景图只是辅助，不是页面内容本身
- 最终逻辑仍然要落在页面结构和属性里
