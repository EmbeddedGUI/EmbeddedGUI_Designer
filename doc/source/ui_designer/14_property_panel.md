# 属性面板

属性面板是你修改控件行为和外观的主入口。绝大多数“这个控件应该长成什么样”，最终都要落到这里。

## 属性面板的定位

它主要负责三类事情：

1. 修改当前选中控件的属性
2. 在多选时做批量字段调整
3. 给出校验、资源引用和相关提示

## 单选和多选的差别

### 单选

单选时，属性面板更像一个完整的控件配置页。你通常会看到：

- 基础信息
- 几何参数
- 文本或样式属性
- 资源绑定
- 回调或用户代码相关入口

### 多选

多选时，属性面板更像“批处理面板”。这时适合做：

- 相同字段的统一设置
- 批量改尺寸或可见性
- 批量绑定同一类行为

## 推荐修改顺序

为了减少返工，建议按下面顺序改：

1. 先改几何参数
2. 再改文本和样式
3. 最后处理资源和行为

这样做的好处是，页面结构还没稳定之前，不会过早在资源和回调上投入太多时间。

## 什么时候优先去属性面板

出现这些需求时，优先看右侧属性面板：

- 文本不对
- 尺寸和位置不对
- 图片或字体没绑定
- 某个控件想换资源
- 想确认当前选中的到底是什么对象

## 属性改了没生效怎么办

优先检查下面几件事：

1. 你是不是选中了正确的控件
2. 这个控件是不是被布局容器接管了位置
3. 当前是不是多选状态
4. 资源路径或字段值是不是无效

## 和其他面板的配合关系

- 结构面板负责“控件属于谁”
- 属性面板负责“控件长什么样”
- 资源面板负责“控件用什么资源”

如果你发现某个问题在属性面板里怎么改都别扭，往往不是属性本身有问题，而是结构没理顺。
