# 导出 C 代码

当你确认页面和资源状态已经稳定，就可以从 Designer 导出 C 代码。

## 入口

菜单位置：

```text
File -> Export C Code...
```

快捷键：

```text
Ctrl+E
```

## 导出前建议先做什么

为了避免把脏状态导出去，建议先：

1. 保存工程
2. 看一眼 Diagnostics
3. 必要时先 `Generate Resources`
4. 如果工程较复杂，先做一次 EXE 预览

## 导出时 Designer 会做什么

导出本质上是把当前工程状态生成成一组 C 文件，并写入你指定目录。

当前仓库特别重要的一点是：

- 导出时会尽量保留用户代码区

## 哪些文件会被反复生成

最需要记住的是这一条：

- 布局生成文件应当视为生成产物，不要在里面长期手工维护业务逻辑

你真正的业务代码，应优先放在保留区或稳定的用户文件中。

## 导出适合哪些场景

- 把页面代码交给 SDK 工程使用
- 做代码审查或人工比对
- 在不打开 Designer 的环境里继续编译

## 导出不是发布

很多新用户会把“导出代码”和“发布 EXE”混为一谈。两者区别是：

- 导出代码：拿到 C 源文件
- Release Build：拿到可分发的程序包

如果你的目标是交付桌面版产物，请继续看 [Release Build](20_release_build.md)。
