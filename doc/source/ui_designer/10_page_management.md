# 页面管理

页面是 Designer 的一级内容单元。一个工程可以只有一个页面，也可以有多个页面。

## 页面入口在哪

页面相关入口主要有两个：

- 左侧 `Project` 面板
- 中央上方的页面标签与导航区

## 常见页面操作

你通常会做下面几件事：

- 新增页面
- 复制页面
- 删除页面
- 重命名页面
- 切换启动页

## 什么时候需要多页面

这些场景很适合拆页面：

- 主界面和设置页分离
- 引导页、主页、详情页分离
- 同一产品有多个固定状态页

如果只是局部弹层或小组件变化，不一定需要新页面。

## 页面切换后的注意点

切换页面后要留意三件事：

1. 当前选中的控件是否已经变化
2. 右侧属性区是否还在显示旧对象
3. 当前编辑的是哪一个页面的 XML

这也是为什么页面命名要清晰，不建议长期保留 `page1`、`page2` 这种临时名字。

## 启动页有什么用

启动页决定应用首次进入时显示哪个页面。多页面工程里，这个设置尤其重要。

建议：

- 主入口页设为启动页
- 调试用页面不要长期保留为启动页

## 页面级配置

除了页面本身的布局，右侧 `Page` 标签页还可以放页面字段、定时器等页面级内容。

如果你发现某个设置不属于单个控件，而属于整页，通常就应该去 `Page` 标签里找。
